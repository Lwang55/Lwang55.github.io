Project Overview

Sewing Guild Website Overview:

For my project I am working with my Client the UNCC Sewing and Cosplay Guild. They are a registered student organization on campus dedicated to encouraging learning how to sew and some of the activities involved such as cosplay. They host large scale events like an annual convention, and want a website that can concisely share information about their events in one consolidated place.

The contents of this site will include a landing page, an About page, Events page, and a Contact page.

Page Design

  • Home
    • Purpose: Welcome to users the website.
    • User: Anyone curious about what the site entails.
    • Content: A picture of the org mascot, as well as a short blurb welcoming the user.
    • User Input? No
    • Data Validation? No
    • Buttons, Hyperlink, Dropdown? Top Navigation.
    • Actions: None
  • About
    • Purpose: Explain the user what the club is about in more detail and outline some of the organization does.
    • User: Anyone who is curious about the club itself and wants to learn more.
    • Content: A paragraph explain the clubs purpose and background.
    • User Input? No
    • Data Validation? No
    • Buttons, Hyperlink, Dropdown? Top Navigation
    • Actions: None
  • FAQ
    • Purpose: Compile some commonly asked questions users may have and provide relevant information.
    • User: Anyone further invested in the club's activites who may have some questions they'd like answered.
    • Content: A series of questions with answers obscured by drop-down menus.
    • User Input? No
    • Data Validation? No
    • Buttons, Hyperlink, Dropdown? Top Navigation, Drop-down menus
    • Actions: Clicking drop-down menus.
  • Events
    • Purpose: Present information about some of the organizations annual events.
    • User: People interested in interacting with the club or wish to attend its events.
    • Content: Image gallery, with short paragraph providing information about the event.
    • User Input? No
    • Data Validation? No
    • Buttons, Hyperlink, Dropdown? Top Navigation, Image gallery
    • Actions:
  • Contact
    • Purpose:
    • User:
    • Content:
    • User Input? Yes, Users input email.
    • Data Validation? Yes, user inputs such as email entry will be checked prior to submission.
    • Buttons, Hyperlink, Dropdown? Top Navigation, Form for user input, Submit Button
    • Actions: Filling out information and pressing submit.

Dynamic Functionality

  • Site-Wide: A navigation bar at top of page will track where the user is on the site, allowing easy navigation and backtracking. Inspired by Build-a-Bear
  • FAQ: The FAQ will have drop-down menus for each question allowing the site to avoid becoming cluttered, an showing only the information needed.
  • Events: The Events page will have an image gallery that the user can scroll through to get a better understanding of what the event looks like. Inspired by Woodelf
  • Contact: A contact information form will be present allowing users to get in touch and leave an email.
